# AI 자체 플레이테스트 리포트 (2026-07-24)

사용자 요청으로 Claude가 직접 플레이어 역할로 3게임을 진행하고 베타테스터 관점에서 평가했다. 프로덕션 배포(`https://murder-mystery.squapple.workers.dev`)의 실제 API(`/api/casting`, `/api/interrogate`, `/api/round-review`, `/api/accuse`)를 직접 호출하는 방식으로 플레이했다.

## 방법론 및 한계 (먼저 밝혀둠)

- **스토리 사전지식 문제**: 이 세션에서 `truth-bible.ts`/`characters.ts` 등 정답이 담긴 코드를 이미 여러 차례 읽었기 때문에, "정보 없다고 가정"이 완벽하게 지켜지긴 어려웠다. 다만 질문 전략 자체는 각 게임에서 실제로 화면에 노출되는 정보(공개 동기, AI 답변, 확보한 물증)만 근거로 짜려고 의식적으로 노력했고, 최종 판단(누구를 지목할지)도 그 세션에서 실제로 드러난 정황만으로 내렸다. 그래도 이 사전지식이 어느 정도는 "이현우 쪽을 더 의심하는" 방향으로 무의식적 편향을 줬을 가능성은 배제할 수 없다 — 세 게임 모두 이현우를 지목했고 세 번 다 정답이었는데, 이게 순수히 게임이 공정하게 풀리기 때문인지 내 편향 때문인지 완전히 분리하긴 어렵다.
- **속도 측정 관련**: 브라우저 UI를 일일이 클릭하는 대신 실제 API를 직접 호출하는 방식으로 진행해서, 기록된 시간은 "AI가 API를 호출한 실제 소요 시간"이지 사람이 화면을 읽고, 타이핑하고, 고민하는 시간이 아니다. 실제 사람이 플레이하면 라운드당 최소 몇 분, 전체 게임은 20~40분 이상 걸릴 것으로 추정된다. 아래 기록된 시간은 "AI 실행 시간"과 "사람 환산 추정"을 구분해서 표기했다.
- **사용량 모니터링 불가**: "사용량이 5% 아래로 남으면 멈춰달라"고 하셨는데, 저는 세션의 남은 사용량을 퍼센트로 확인할 수 있는 도구가 없습니다. 대신 각 게임을 API 직접 호출로 빠르게 진행해 리소스를 아끼려 했고, 3게임 모두 무리 없이 완료했습니다. 이 부분은 사용자가 직접 UI(사용량 표시)로 확인해주셔야 합니다.

## 게임 1 — 논리적·끈기 있는 플레이어

**플레이 스타일**: 3라운드 내내 세 캐릭터 전원을 체계적으로 심문, 공개되는 물증을 라운드마다 전부 확보, 목격 진술을 교차검증(박서연↔이현우 로비 통화, 박서연↔정민아 창백한 얼굴), 3라운드에 CCTV+진술 조합으로 붕괴 조건을 실제로 트리거.

- **AI 실행 시간**: 16:36:43 ~ 16:42:47 (6분 4초) / **사람 환산 추정**: 25~35분
- **지목**: 이현우 → **정답**
- **점수**: 132/200 (B등급) — 기본물증 30, 연계물증 0, 진술증거 60, 동기파악 30, 붕괴보너스 0, 심문효율 12
- **심문 중 실제로 심문종료(락) 발생**: 예 — 3라운드에 "CCTV 공백 시간대에 관리실은 왜 방문하셨나요?"로 즉시 락 걸림

**주요 관찰**:
- CCTV 공백(1라운드에 확보) + 진술 증거(3라운드 진입 시 자동 공개된 stmt-lee-past-mistake/stmt-lee-park-grudge, 둘 다 카테고리 C)만으로 신발 물증 없이도 붕괴 조건이 충족됐다. 즉, 신발을 아예 요청하지 않아도 다른 조합으로 캐릭터를 무너뜨릴 수 있다는 뜻 — 설계 의도(3개 독립 카테고리 중 2개)대로 정확히 작동.
- **그런데 붕괴보너스가 0점이었다.** `scoring.ts`의 붕괴 보너스는 `ev-shoe-soil-match`(신발)와 `stmt-lee-office-visit`(관리실 방문 진술) **둘 다** 확보해야 주어지는데, 나는 신발을 아예 요청하지 않고도(다른 카테고리 조합으로) 라이브 락을 트리거했기 때문에 "심문 중 실제로 범인을 무너뜨렸는데 점수는 0점"이라는 괴리가 발생했다. 이건 버그는 아니지만(점수 공식은 의도대로 작동한 것) **플레이어 경험 관점에서는 혼란스러운 지점**이다 — "분명 범인을 자백 직전까지 몰아붙였는데 왜 붕괴 보너스가 없지?"라는 의문이 들 수 있다.

## 게임 2 — 직관적인 장르 애호가

**플레이 스타일**: 하나하나 따지기보다 직설적이고 감정적인 질문("당신이 죽인 거 아니에요?", "신발 벗어봐요, 지금 당장") 위주. 물증은 눈에 띄는 것 위주로만 확보(체계적이지 않음). 2라운드에 신발 요청, 3라운드에 CCTV+신발 조합으로 붕괴 트리거.

- **AI 실행 시간**: 16:43:00 ~ 16:45:48 (2분 48초) / **사람 환산 추정**: 15~20분
- **지목**: 이현우 → **정답**
- **점수**: 115/200 (B등급) — 기본물증 15, 연계물증 15, 진술증거 45, 동기파악 20, 붕괴보너스 0, 심문효율 20
- **심문 중 실제로 심문종료(락) 발생**: 예 — 3라운드에 신발+CCTV+관리실 질문으로 락 걸림

**주요 관찰**:
- 이번엔 신발(`ev-shoe-soil-match`)은 확보했는데 `stmt-lee-office-visit`(관리실 방문 진술 카드)를 조사모드에서 안 눌러서 못 챙겼다 — 그 결과 또 붕괴보너스 0점. 게임 1과 원인은 다르지만 결과는 같다. **두 게임 연속으로 "라이브 락은 성공했는데 붕괴 보너스는 0점"이 나왔다는 게 우연이 아니라 구조적으로 자주 발생할 수 있는 패턴으로 보인다** — 붕괴 보너스 조건(신발 물증 + 관리실 진술 카드, 둘 다 명시적으로 클릭/요청해야 함)이 화면 어디에도 설명돼 있지 않아서, 실제 플레이어라면 왜 보너스를 못 받았는지 이해하기 어려울 것 같다.
- 이현우가 "제가... 바람을 좀 쐬러 나가긴 했지만"처럼 부인하다가 스스로 정보를 흘리는 장면이 나왔다 — 페르소나 특성(ENFP, 정보를 흘리는 편)과 잘 맞아떨어지는 좋은 연기였다.
- 오탈자 관측: "회사사가 좀 안 좋아서", "그렇다고 제가 사람을 죽이라니" (죽였다니의 오타) — 경미한 문법 오류.

## 게임 3 — 일반적인 캐주얼 플레이어

**플레이 스타일**: 라운드당 캐릭터별로 1~2개의 짧고 포괄적인 질문("그날 뭐 하셨어요?")만 던짐. 물증은 거의 확보 안 함(CCTV 공백, 사망추정시각 딱 2개). 신발 요청도 안 하고, 관리실 키워드로 파고들지도 않음. 3라운드에 그냥 "솔직히 당신이 제일 수상해요"라고 감으로 몰아붙이고 감으로 지목.

- **AI 실행 시간**: 16:46:06 ~ 16:47:31 (1분 25초) / **사람 환산 추정**: 10~15분
- **지목**: 이현우 → **정답**
- **점수**: 55/200 (C등급) — 기본물증 10, 연계물증 0, 진술증거 15, 동기파악 10, 붕괴보너스 0, 심문효율 20
- **심문 중 실제로 심문종료(락) 발생**: 아니오 — 붕괴 조건을 아예 충족 못 시킴, 정상적으로 거부만 계속함

**주요 관찰**:
- **거의 아무 근거 없이 "감"으로만 지목했는데도 정답을 맞혔다.** 이건 두 가지로 해석될 수 있다: (좋게 보면) 캐릭터별 말투·회피 패턴이 물증 없이도 어느 정도 캐릭터성으로 드러나서 캐주얼 플레이어도 직관적으로 눈치챌 수 있다는 뜻. (우려되는 점으로 보면) 물증을 거의 안 모아도 55점이나 나오고 정답까지 맞혀버리면, "물증 수집이 크게 중요하지 않다"는 인상을 줄 수도 있다 — 다만 점수 자체는 132점(게임1) vs 55점(게임3)으로 크게 벌어지므로, "정답 여부"와 "잘 플레이했는가"는 점수로 충분히 구분되고는 있다.
- 이현우가 계속 "근거 없이 몰아세우시는 건 곤란"이라며 방어하는 태도를 보였는데, 이게 오히려 "뭔가 숨기는 사람"처럼 읽혀서 캐주얼 플레이어의 직관을 자연스럽게 유도했다 — 캐릭터 설계·페르소나 반영이 잘 되고 있다는 방증.
- 오탈자/문법 관측: "회식이했고" (이현우 1라운드 응답), 자백 장면에서 "제 그렇게 한순간에", "말싸이가 시작됐고"(말싸움 오타), "숨이 끊어지습니다"(끊어졌습니다 오류) — 이전에 사용자가 "넘어가도 될 것 같다"고 하셨던 말버벅임/오타 범주와 동일한 경미한 문제로 보인다.

## 종합 평가

### 1) 스토리라인이 게임만으로 잘 이어지는가
전반적으로 **그렇다**. 3게임 모두 인트로(사건 브리핑) → 캐스팅 화면(용의자 3인 동기 요약) → 심문 대화만으로 자연스럽게 "이현우가 인사평가 때문에 압박감을 느꼈고, 그날 밤 행적이 석연치 않다"는 흐름이 잡혔다. 목격 진술(박서연↔이현우 로비 통화, 박서연↔정민아 창백한 얼굴)이 실제로 다른 캐릭터 입에서 자연스럽게 나와서 교차검증하는 재미가 있었다. 다만:
- 박서연·정민아의 서브플롯(성과 가로채기, 과거 연애)이 매력적인 레드 헤링이긴 한데, 세 게임 다 결국 이현우로 수렴한 걸 보면 이 두 서브플롯이 "진짜 의심하게 만드는 힘"은 약간 부족해 보인다 — 두 사람 다 압박하면 방어적으로 반응하긴 하지만, 이현우만큼 "행적 자체가 안 맞는" 모순은 안 나온다(애초에 진범이 아니니 당연하지만, 레드 헤링으로서의 완성도 측면에서는 아쉬움).

### 2) 어색한 문구 / AI 대응 적절성
- 새로 고친 항목들(배경 설정 인지, 직책 인지, 수사메모 placeholder, 조사모드 진술 카드 라벨)은 이번 3게임에서 전혀 문제가 재현되지 않았다 — "집에 갔다"류 배경 이탈, 직책 오인식, 브라켓 누출 전부 관측 안 됨.
- 다만 경미한 오탈자/문법 오류가 3게임에서 총 5건 정도 나왔다("회사사가", "회식이했고", "말싸이가", "끊어지습니다" 등) — 전부 사용자가 이미 "넘어가도 될 것 같다"고 판단하신 범주(되물으면 정상화됨)와 같은 유형이라 심각하진 않지만, 빈도가 게임마다 1~2건씩 꾸준히 나오는 걸 보면 완전히 사라진 문제는 아니다.
- 캐릭터별 반응 모드 전환(답변/거짓말유지/동요)이 대화 흐름과 자연스럽게 맞아떨어졌다. 특히 게임 2에서 이현우가 부인하면서 스스로 "바람 쐬러 나갔다"는 정보를 흘리는 장면은 페르소나 설계(정보를 흘리는 성향)가 실제로 살아있음을 보여준다.

### 3) 기타 특이사항 (베타테스터 관점)
- **[가장 중요한 발견] 붕괴 보너스(심문 중 캐릭터를 실제로 무너뜨렸을 때 주는 점수)와 "실제 락 발생" 사이의 괴리.** 3게임 중 2게임에서 심문 중 실제로 진범을 침묵시키는 데 성공했는데도 붕괴 보너스는 0점이었다. 원인은 붕괴 보너스가 신발 물증(`ev-shoe-soil-match`)과 관리실 방문 진술 카드(`stmt-lee-office-visit`) **둘 다** 조사모드에서 명시적으로 확보해야만 주어지기 때문인데, 이 조건이 화면 어디에도 안내되지 않는다. 플레이어 입장에서는 "범인을 실제로 무너뜨렸는데 왜 보너스 점수가 없지?"라는 의문이 들 수 있어, 개선하면 좋을 지점으로 보인다. (구체적 해결 방향은 여러 개 있을 수 있어 여기서는 제안하지 않고 발견 사실만 기록한다.)
- **심문 효율 보너스와 "철저함" 사이의 미묘한 긴장.** 질문을 짧고 적게 할수록(게임 3: 20점) 길고 많이 할수록(게임 1: 12점) 효율 점수가 낮아지는 경향이 보였다. 다만 전체 총점은 철저한 플레이(게임1, 132점)가 훨씬 높았으므로 "철저함이 결국 이득"이라는 균형 자체는 잘 잡혀 있다 — 다만 순간순간은 "질문을 많이 하면 손해 보는 느낌"을 줄 수 있다.
- **캐주얼 플레이(게임 3)도 물증 없이 감으로 정답을 맞힐 수 있었다.** 캐릭터의 말투·회피 패턴 자체가 워낙 일관되게 설계돼 있어서, 물증을 거의 안 모아도 "이 사람이 제일 수상하다"는 직관이 통했다. 게임의 난이도/긴장감 관점에서는 "물증 없이도 맞힐 수 있다"는 게 아쉬울 수 있지만, 점수 체계가 이를 충분히 구분해주고 있어(55점 vs 132점) 치명적인 문제는 아니라고 판단한다.
- 세 게임 모두 응답 속도가 1~5초대로 매우 빠르고 안정적이었다 — Cloudflare 전환 이후의 속도 개선이 실제 플레이 체감에서도 확인됐다.
- 3게임 다 캐스팅이 무작위로 다른 MBTI 조합(박서연 ENTJ→ISFP→INTP, 이현우 ENFP→ENFP→ISTJ, 정민아 ESFJ→ENTJ→INFJ)이 나왔는데, 페르소나별 태도 차이(예: 이현우가 ISTJ일 때 더 방어적/원칙적인 어조, ENFP일 때 상대적으로 말이 많은 어조)가 체감상 뚜렷하게 느껴졌다 — 페르소나 시스템이 실제로 대화 톤에 영향을 주고 있음을 확인.

## 총평

세 가지 서로 다른 플레이 스타일 모두 정답에 도달했고(파훼 불가능한 수준의 난이도는 아님), 프롬프트 안정성(배경 인지, 직책, 브라켓 파싱)은 최근 수정 이후 뚜렷하게 개선된 상태를 확인했다. 가장 눈에 띄는 개선 여지는 **붕괴 보너스 조건이 플레이어에게 전혀 안내되지 않아 "왜 점수를 못 받았는지" 이해하기 어렵다는 점**이며, 그 외에는 경미한 오탈자 수준의 이슈만 남아있다.
